# Project Charter

## Mission

GhostLink exists to support private digital communication through open-source,
auditable and self-hostable software.

## Initial product objective

The first release target is intentionally constrained:

- one relay;
- two clients;
- one encrypted text message;
- plaintext available only on sender and recipient devices.

## Values

- digital freedom;
- cypherpunk culture;
- user sovereignty;
- open source;
- transparency;
- auditability;
- minimal trust.

## Non-goals for the first milestone

- anonymity against a global network observer;
- group messaging;
- voice or video;
- file transfers;
- multi-device synchronization;
- mobile clients;
- production-readiness.

## Quality rule

Working code is not sufficient. A change must be tested, documented and compatible
with the threat model before it is considered complete.

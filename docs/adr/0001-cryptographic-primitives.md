# ADR-0001: Initial cryptographic primitives

- Status: Accepted for proof of concept
- Date: 2026-07-10

## Context

GhostLink requires a minimal, reviewable cryptographic proof of concept without
inventing algorithms.

## Decision

Use libsodium through PyNaCl:

- X25519-compatible `Box` key agreement and authenticated public-key encryption;
- Curve25519 key pairs generated locally;
- cryptographically secure random nonces generated by libsodium.

## Consequences

Advantages:

- mature and widely reviewed primitives;
- small API surface;
- safe nonce generation through the library.

Limitations:

- this is not a complete messaging protocol;
- no Double Ratchet;
- no forward secrecy across message history;
- identity and ephemeral keys are not yet separated.

The module must be replaced or extended by a formally specified session protocol
before production use.

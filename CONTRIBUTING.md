# Contributing

## Workflow

1. Create an issue describing the change.
2. Create a branch named `feat/...`, `fix/...`, `docs/...`, or `refactor/...`.
3. Add or update tests.
4. Run all quality checks.
5. Open a pull request.

## Commit convention

Use Conventional Commits:

- `feat:`
- `fix:`
- `docs:`
- `test:`
- `refactor:`
- `chore:`
- `security:`

## Definition of done

A change is complete only when it is:

- implemented;
- tested;
- typed;
- documented;
- reviewed;
- compatible with the threat model.
